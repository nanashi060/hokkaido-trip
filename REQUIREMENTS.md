# 北海道旅行サイト 要件定義書

## 1. 概要

2026年7月13日(月)〜7月17日(金)の4泊5日、二人で行く北海道旅行(札幌・小樽・函館)の旅程を
ビジュアライズする Web サイト。「見やすい」だけでなく「見ててワクワクする」体験を提供する。

- **旅行日程**: 2026/7/13 〜 2026/7/17(4泊5日)
- **訪問エリア**: 札幌・小樽・函館
- **利用者**: 本人と彼女の2人(スマホ・PC 両方から閲覧)

## 2. 目的

1. 旅行前:予定を二人で眺めてワクワクしながら共有・確認できる
2. 旅行中:今日どこに行くか、次のスポットの情報(営業時間・場所)をすぐ確認できる
3. 旅行後:思い出のアーカイブとして残せる(将来拡張)

## 3. 機能要件

### 3.1 日別タイムライン(旅程ビュー)

- Day 1〜Day 5 のタブ or セクション切り替え
- 各日の予定を時系列で縦に並べたタイムライン表示
  - 時刻・スポット名・カテゴリアイコン(食事 / 観光 / 移動 / 宿泊 など)
  - 移動手段(JR・徒歩・タクシー等)と所要時間を予定の間に表示
- 「今日」の日程を自動でハイライト(旅行当日に開いたとき)

### 3.2 地図ビュー

- **Leaflet + OpenStreetMap** を使用(API キー不要・無料)
- 全スポットをカテゴリ別アイコンのピンで表示
- 日ごとに色分けし、訪問順をルート線(ポリライン)で結ぶ
- ピンクリックでスポット詳細カードへリンクするポップアップ
- 日別フィルタ(Day 1 だけ表示、など)

### 3.3 スポット詳細カード

各スポットについて以下を表示:

- 写真(1枚以上)
- スポット名・キャッチコピー
- カテゴリ(グルメ / 絶景 / 体験 / 宿 / 移動拠点)
- 営業時間・定休日
- 予算目安
- メモ(予約状況、注文したいメニュー、注意点など)
- 外部リンク(公式サイト・食べログ・Google マップ)

### 3.4 ワクワク演出(デザイン要件)

- ポップで旅のしおり感のあるカラフルなデザイン
- スクロール連動アニメーション(タイムラインが順に現れる等)
- 出発日までのカウントダウン表示(ヒーローセクション)
- Day ごとのテーマカラー
- 絵文字・アイコンを効果的に使用
- ページ読み込み時の軽いアニメーション(飛行機が飛ぶ、など)

## 4. 非機能要件

| 項目 | 内容 |
|------|------|
| レスポンシブ | スマホファースト(旅行中はスマホで見る)、PC でも快適 |
| パフォーマンス | 画像は遅延読み込み。3G 回線でも実用的な速度 |
| オフライン耐性 | 旅行中の電波が弱い場所を考慮し、依存ライブラリは最小限 |
| ビルド | 不要(素の HTML/CSS/JS)。push すれば即デプロイ |
| ブラウザ | iOS Safari / Android Chrome / PC Chrome 最新版 |

## 5. 技術スタック

| 領域 | 選定 | 理由 |
|------|------|------|
| フロントエンド | 素の HTML / CSS / JavaScript | ビルド不要、GitHub Pages に直接デプロイ可 |
| データ | JSON ファイル(`data/` 配下) | スポット追加=JSON 追記のみ。コード変更不要 |
| 地図 | Leaflet + OpenStreetMap | API キー不要・無料・カスタマイズ自由 |
| ホスティング | GitHub Pages | 無料・URL を彼女に共有できる |
| リポジトリ | GitHub(`hokkaido-trip`) | バージョン管理・Pages 連携 |

## 6. データ設計(拡張性の要)

**「行く場所が増えても問題ない」を実現する中核。** サイトの表示はすべて JSON データから
動的に生成し、スポット追加時に HTML / JS の変更を不要にする。

### 6.1 `data/trip.json` — 旅行全体のメタ情報

```json
{
  "title": "北海道旅行 2026",
  "startDate": "2026-07-13",
  "endDate": "2026-07-17",
  "days": [
    { "day": 1, "date": "2026-07-13", "area": "札幌", "theme": "#FF6B6B", "title": "札幌到着!" }
  ]
}
```

### 6.2 `data/spots.json` — スポットマスタ(追加はここに追記するだけ)

```json
{
  "spots": [
    {
      "id": "otaru-canal",
      "name": "小樽運河",
      "category": "sightseeing",
      "area": "小樽",
      "lat": 43.198,
      "lng": 141.002,
      "photo": "images/otaru-canal.jpg",
      "catchcopy": "ノスタルジックな夜景の名所",
      "hours": "散策自由(ライトアップは日没〜24:00)",
      "budget": "無料",
      "memo": "夕方に行くのがおすすめ",
      "links": { "official": "https://..." }
    }
  ]
}
```

### 6.3 `data/itinerary.json` — 日別スケジュール(スポット ID を参照)

```json
{
  "schedule": [
    {
      "day": 1,
      "items": [
        { "time": "10:30", "spotId": "otaru-canal", "note": "" },
        { "time": "12:00", "type": "move", "method": "JR", "duration": "40分", "note": "小樽→札幌" }
      ]
    }
  ]
}
```

### 6.4 拡張ルール

- **スポット追加**: `spots.json` に 1 エントリ追記 + `itinerary.json` に参照追加のみ
- **日程追加**(延泊等): `trip.json` の `days` に追記すれば Day タブが自動で増える
- **カテゴリ追加**: JS 側のカテゴリ定義(アイコン・色のマップ)に 1 行追加
- **エリア追加**(旭川に行くことになった等): データに書くだけ。地図は自動でフィット
- 将来拡張の余地: 旅行後の写真アルバム化、費用記録、次の旅行(別 JSON セット)への横展開

## 7. 画面構成

```
┌─ ヒーロー ─────────────────────┐
│ タイトル / 日程 / カウントダウン        │
├─ 日別タイムライン ──────────────┤
│ [Day1][Day2][Day3][Day4][Day5] タブ    │
│  ↓ 時系列タイムライン(スポットカード) │
├─ 地図 ─────────────────────┤
│ Leaflet 地図(日別色分け・ルート線)    │
├─ スポット一覧 ─────────────────┤
│ カテゴリフィルタ + カードグリッド       │
└─ フッター ─────────────────────┘
```

シングルページ構成(セクションスクロール)。ナビゲーションはスティッキーヘッダー。


## 9. 留意事項

- GitHub Pages(無料プラン)は**公開リポジトリ**が必要。ホテル名・日程など個人情報に
  近い内容が公開される点は了承の上で運用する(氏名・予約番号等は載せない)
- 写真は容量に注意(1 枚 500KB 目安に圧縮)
